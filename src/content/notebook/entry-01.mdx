Standard scaled dot-product attention computes, for queries $Q$, keys $K$,
and values $V$:

$$
\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

The $QK^\top$ term is the bottleneck<Sidenote id="1">This single matrix
multiply dominates both FLOPs and peak memory once sequence length $n$
exceeds a few thousand tokens, since it produces an $n \times n$ score
matrix.</Sidenote>: it scales quadratically with sequence length $n$,
giving the familiar $O(n^2 d_k)$ complexity for a single attention head.

## Where sparsity helps

Most of the mass in a softmax-normalized attention row concentrates on a
small number of keys<Sidenote id="2">Empirically, over long contexts the
top ~5–10% of keys by score account for the large majority of the
softmax probability mass.</Sidenote>. If we can cheaply predict which
key blocks matter for a given query *before* running the full dense
score computation, we only need to materialize scores for those blocks.

For a block size $b$ and $k$ selected blocks per query, the effective
complexity becomes:

$$
O\!\left(n \cdot k \cdot b \cdot d_k\right) \quad \text{instead of} \quad O(n^2 d_k)
$$

which is sub-quadratic whenever $k \cdot b \ll n$.
